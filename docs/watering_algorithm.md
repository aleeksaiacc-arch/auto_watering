# Документация алгоритма системы полива растений

## Обзор
Система полива растений — это контроллер автоматического орошения на базе Arduino, который отслеживает влажность почвы и управляет водяным насосом. Работает в двух режимах (AUTO и MANUAL) и взаимодействует с модулем ESP8266 для подключения к MQTT.

## Константы

### Определения пинов оборудования
- `MOISTURE_PIN`: Аналоговый пин для датчика влажности (A0)
- `PUMP_PIN`: Цифровой пин для управления реле насоса (4)
- `ESP_RX_PIN`: Цифровой пин для RX ESP8266 (SoftwareSerial RX) (10)
- `ESP_TX_PIN`: Цифровой пин для TX ESP8266 (SoftwareSerial TX) (11)

### Константы порогов
- `THRESHOLD_DRY`: Порог сухой почвы по умолчанию в процентах (30%)
  - Когда влажность падает ниже этого значения, насос активируется в режиме AUTO
- `THRESHOLD_WET`: Порог влажной почвы в процентах (70%)
  - Когда влажность достигает этого значения, насос останавливается в режиме AUTO

### Константы управления насосом
- `PUMP_DURATION_MS`: Длительность работы насоса по умолчанию (10000 мс = 10 секунд)
  - Минимальное время работы насоса перед повторной проверкой влажности
- `PUMP_MAX_MS`: Максимально допустимое время работы насоса (60000 мс = 60 секунд)
  - Предел безопасности для предотвращения повреждения насоса или затопления
  - Применяется в обоих режимах AUTO и MANUAL

### Константы датчика
- `SENSOR_READ_INTERVAL`: Интервал между показаниями датчика (5000 мс = 5 секунд)
- `SENSOR_SAMPLES`: Количество аналоговых выборок для усреднения (5)
  - Уменьшает шум в показаниях влажности

### Константы калибровки ADC
- `ADC_DRY`: Значение ADC, представляющее полностью сухую почву (3500)
  - Более высокое значение ADC = более низкая проводимость = более сухая почва
- `ADC_WET`: Значение ADC, представляющее полностью влажную почву (800)
  - Более низкое значение ADC = более высокая проводимость = более влажная почва

### Буферы и флаги функций
- `CMD_BUFFER_SIZE`: Размер буфера ввода команд (32 байта)
- `USE_ESP8266`: Включить коммуникацию ESP8266 через SoftwareSerial (1 = включено)
- `USE_SERIAL_DEBUG`: Включить отладочный вывод в последовательный порт (0 = отключено)

### Публикация MQTT
- `MQTT_PUBLISH_INTERVAL`: Минимальный интервал между обновлениями статуса MQTT (60000 мс = 60 секунд)

## Перечисления

- `Mode`: Режим работы
  - `MODE_AUTO`: Автоматический полив на основе порогов влажности
  - `MODE_MANUAL`: Ручное управление насосом через команды
- `PumpState`: Текущее состояние насоса
  - `PUMP_OFF`: Насос неактивен
  - `PUMP_ON`: Насос активен

## Глобальные переменные состояния

- `currentMode`: Текущий режим работы (по умолчанию: `MODE_AUTO`)
- `pumpState`: Текущее состояние насоса (по умолчанию: `PUMP_OFF`)
- `thresholdDry`: Динамический порог сухости (инициализируется из `THRESHOLD_DRY`, может быть изменен через команду)
- `thresholdWet`: Порог влажности (инициализируется из `THRESHOLD_WET`, фиксированный)
- `pumpStartTime`: Временная метка последней активации насоса (миллисекунды)
- `lastSensorRead`: Временная метка последнего показания датчика (миллисекунды)
- `lastMqttPublish`: Временная метка последней публикации MQTT (миллисекунды)
- `lastMoisturePercent`: Последний вычисленный процент влажности (-1 = не инициализирован)
- `cmdBuffer[CMD_BUFFER_SIZE]`: Буфер для накопления символов команд последовательного порта
- `cmdIndex`: Текущий индекс в буфере команд (0-31)

## Настройка последовательной коммуникации

- **Основной последовательный порт**: Аппаратный последовательный порт на скорости 115200 бод (для отладки, если включено)
- **Последовательный порт ESP8266**: SoftwareSerial на пинах 10 (RX) и 11 (TX) на скорости 115200 бод
- **Макрос**: `WIFI_SERIAL` указывает на последовательный порт ESP8266, когда `USE_ESP8266` включен, иначе на аппаратный последовательный порт

## Алгоритм инициализации (`setup`)

1. **Конфигурация пинов**:
   - Устанавливает `MOISTURE_PIN` как INPUT (аналоговый)
   - Устанавливает `PUMP_PIN` как OUTPUT (цифровой)
   - Инициализирует насос в состояние LOW (выключен)

2. **Аналоговая опорная точка**: Устанавливает в DEFAULT (5В для большинства плат Arduino)

3. **Инициализация последовательного порта**:
   - Инициализирует аппаратный последовательный порт на скорости 115200 бод
   - Если `USE_ESP8266` включен: Инициализирует SoftwareSerial на скорости 115200 бод

4. **Загрузка из EEPROM**: Загружает сохраненный порог из EEPROM (если валиден)

## Алгоритм хранения в EEPROM

### Загрузка порога (`loadThresholdFromEeprom`)
- **Формат хранения**: 16-битное целое число, хранимое в двух байтах
  - Байт 0: Старший байт `(thresholdDry >> 8) & 0xFF`
  - Байт 1: Младший байт `thresholdDry & 0xFF`
- **Чтение**: Объединяет байты: `(EEPROM.read(0) << 8) | EEPROM.read(1)`
- **Проверка**: Загружает только если значение между 1 и 99 (валидный диапазон процентов)
- **По умолчанию**: Если невалидно, использует константу `THRESHOLD_DRY`

### Сохранение порога (`saveThresholdToEeprom`)
- **Запись**: Разделяет 16-битное значение на два байта
  - Записывает старший байт по адресу 0
  - Записывает младший байт по адресу 1
- **Вызывается когда**: Порог изменен через команду `SET_THRESHOLD`

## Алгоритм датчика влажности

### Сырое показание (`readMoistureRaw`)
1. **Усреднение**: Берет `SENSOR_SAMPLES` (5) последовательных показаний
2. **Тайминг**: Задержка 10 мс между выборками (позволяет стабилизироваться ADC)
3. **Вычисление**: Возвращает целочисленное среднее всех выборок
4. **Назначение**: Уменьшает шум и обеспечивает стабильные показания

### Преобразование в проценты (`moistureToPercent`)
1. **Маппинг**: Использует функцию Arduino `map()`
   - Маппит из диапазона ADC `[ADC_DRY, ADC_WET]` в диапазон процентов `[0, 100]`
   - Формула: `map(raw, 3500, 800, 0, 100)`
2. **Ограничение**: Ограничивает результат до `[0, 100]` используя `constrain()`
3. **Обратная зависимость**: Более высокий ADC = более сухая почва = более низкий процент
   - Сухая (3500 ADC) → 0%
   - Влажная (800 ADC) → 100%

## Алгоритм управления насосом (`setPump`)

### Включение насоса
1. **Проверка состояния**: Если насос в настоящее время выключен, записывает время начала (`pumpStartTime = millis()`)
2. **Проверка безопасности**: Вычисляет прошедшее время с момента запуска насоса
3. **Временной лимит**: Если прошедшее время < `PUMP_MAX_MS` (60 секунд):
   - Устанавливает пин в HIGH (активирует насос)
   - Обновляет состояние в `PUMP_ON`
4. **Аварийное отключение**: Если прошедшее время >= `PUMP_MAX_MS`:
   - Устанавливает пин в LOW (деактивирует насос)
   - Обновляет состояние в `PUMP_OFF`
   - **Назначение**: Предотвращает повреждение насоса от длительной работы

### Выключение насоса
1. Устанавливает пин в LOW (деактивирует насос)
2. Обновляет состояние в `PUMP_OFF`
3. **Примечание**: Не сбрасывает `pumpStartTime` (сохраняется для следующей активации)

## Алгоритм автоматического режима (`runAutoMode`)

### Когда насос включен
1. **Вычисление прошедшего времени**: `elapsed = millis() - pumpStartTime`
2. **Условия остановки** (любое условие останавливает насос):
   - **По времени**: `elapsed >= PUMP_DURATION_MS` (минимум 10 секунд)
   - **По влажности**: `moisturePercent >= thresholdWet` (достигнуто 70%)
3. **Действие**: Если любое условие выполнено:
   - Вызывает `setPump(false)` для остановки насоса
   - Отправляет обновление статуса через `sendStatus()`
4. **Ранний возврат**: Выходит из функции (насос продолжает работать, если условия не выполнены)

### Когда насос выключен
1. **Проверка сухой почвы**: Если `moisturePercent < thresholdDry` (ниже 30%):
   - Вызывает `setPump(true)` для запуска насоса
   - Отправляет обновление статуса через `sendStatus()`
2. **Бездействие**: Если влажность выше порога, ничего не делает

**Принцип**: Управление с гистерезисом - использует разные пороги для активации (сухость) и деактивации (влажность) для предотвращения быстрого переключения.

## Алгоритм обработки команд (`processCommand`)

Обрабатывает текстовые команды, полученные через последовательный порт. Все команды вызывают обновление статуса, кроме случаев, указанных отдельно.

### Команда: `"PUMP_ON"`
1. Переключает режим в `MODE_MANUAL`
2. Активирует насос через `setPump(true)`
3. Отправляет обновление статуса

### Команда: `"PUMP_OFF"`
1. Деактивирует насос через `setPump(false)`
2. Переключает режим в `MODE_AUTO`
3. Отправляет обновление статуса

### Команда: `"GET_STATUS"`
1. Отправляет только обновление статуса (без изменения состояния)

### Команда: `"SET_THRESHOLD:<value>"`
1. Извлекает числовое значение после `"SET_THRESHOLD:"` (14 символов)
2. Проверяет, что значение между 1 и 99
3. Если валидно:
   - Обновляет переменную `thresholdDry`
   - Сохраняет в EEPROM через `saveThresholdToEeprom()`
   - Отправляет обновление статуса
4. Если невалидно: Игнорирует команду (без действий)

### Команда: `"MODE_AUTO"`
1. Переключает `currentMode` в `MODE_AUTO`
2. Отправляет обновление статуса
3. **Примечание**: Не останавливает насос, если он работает (насос продолжает работать до выполнения условий)

### Команда: `"MODE_MANUAL"`
1. Переключает `currentMode` в `MODE_MANUAL`
2. Отправляет обновление статуса
3. **Примечание**: Не изменяет состояние насоса

## Алгоритм отправки статуса (`sendStatus`)

Генерирует и отправляет полный статус системы через последовательный порт.

1. **Чтение датчика**: Получает свежее показание влажности
   - Вызывает `readMoistureRaw()` для текущего значения ADC
   - Преобразует в проценты через `moistureToPercent()`
   - Обновляет `lastMoisturePercent`

2. **Форматирование сообщения**: Создает строку статуса
   - Формат: `"MOISTURE:%d|PUMP:%s|MODE:%s|THRESHOLD:%d\n"`
   - Значения:
     - Влажность: Текущий процент (0-100)
     - Насос: "ON" или "OFF" на основе `pumpState`
     - Режим: "AUTO" или "MANUAL" на основе `currentMode`
     - Порог: Текущее значение `thresholdDry`

3. **Отправка**: Передает через `WIFI_SERIAL` (ESP8266 или Serial)

4. **Отладочный вывод**: Если `USE_SERIAL_DEBUG` включен, также выводит в аппаратный последовательный порт с префиксом "-> "

**Вызывается когда**:
- Изменяется состояние насоса
- Изменяется режим
- Изменяется порог
- Получена команда `GET_STATUS`
- Автоматический режим останавливает насос

## Опрос команд последовательного порта (`pollSerial`)

Читает и обрабатывает команды из последовательного ввода.

1. **Чтение посимвольно**:
   - Читает все доступные символы из `WIFI_SERIAL`
   - Для каждого символа:
     - **Обнаружение новой строки** (`\n` или `\r`):
       - Если в буфере есть содержимое (`cmdIndex > 0`):
         - Добавляет нулевое завершение буфера
         - Обрабатывает команду через `processCommand()`
         - Сбрасывает индекс буфера в 0
     - **Накопление символов**:
       - Если в буфере есть место (`cmdIndex < CMD_BUFFER_SIZE - 1`):
         - Сохраняет символ в `cmdBuffer[cmdIndex]`
         - Увеличивает `cmdIndex`
     - **Защита от переполнения**: Символы за пределами размера буфера отбрасываются

**Принцип**: Протокол на основе строк - команды должны заканчиваться символом новой строки.

## Публикация MQTT (Условная, `USE_ESP8266`)

### Периодическая публикация (`mqttPublish`)
1. **Предусловия**:
   - `lastMoisturePercent >= 0` (датчик был прочитан)
   - `millis() - lastMqttPublish >= MQTT_PUBLISH_INTERVAL` (прошло 60 секунд)
2. **Обновление временной метки**: Устанавливает `lastMqttPublish = millis()`
3. **Публикация топиков**:
   - `"plant/moisture"`: Текущий процент влажности как строка
   - `"plant/pump"`: "ON" или "OFF" на основе состояния насоса
   - `"plant/mode"`: "AUTO" или "MANUAL" на основе текущего режима

**Назначение**: Периодические обновления статуса в MQTT брокер (независимо от изменений статуса).

### Публикация топика (`mqttPublishTopic`)
1. **Формат**: Отправляет `"MQTT_PUB:<topic>:<payload>\n"` на ESP8266
2. **Обработка ESP8266**: Мост ESP8266 парсит и публикует в MQTT брокер
3. **Назначение**: Позволяет Arduino публиковать произвольные MQTT сообщения

## Алгоритм основного цикла (`loop`)

Основной цикл выполнения с тремя основными обязанностями:

### 1. Опрос команд
- Вызывает `pollSerial()` для обработки любых входящих команд
- **Неблокирующий**: Обрабатывает все доступные символы, затем продолжает

### 2. Периодическое чтение датчика и управление
- **Проверка тайминга**: Если `millis() - lastSensorRead >= SENSOR_READ_INTERVAL` (5 секунд):
  1. **Обновление временной метки**: Устанавливает `lastSensorRead = now`
  2. **Чтение датчика**: Получает сырое значение ADC и преобразует в проценты
  3. **Управление в зависимости от режима**:
     - **Режим AUTO**: Вызывает `runAutoMode(pct)` для автоматического управления насосом
     - **Режим MANUAL**: Если насос включен, проверяет таймаут безопасности
       - Если `elapsed >= PUMP_MAX_MS`: Принудительно выключает насос (предел безопасности)
  4. **Обнаружение изменения влажности**: Если `pct != lastMoisturePercent`:
     - Обновляет `lastMoisturePercent`
     - Отправляет обновление влажности: `"MOISTURE:%d\n"` на ESP8266
     - **Назначение**: Немедленное уведомление об изменениях влажности

### 3. Публикация MQTT (если включено)
- Если `USE_ESP8266` включен: Вызывает `mqttPublish()`
- **Ограничено**: Публикует только если прошло 60 секунд с последней публикации

## Временные характеристики

- **Интервал чтения датчика**: Каждые 5 секунд
- **Интервал публикации MQTT**: Минимум 60 секунд между публикациями
- **Минимальная длительность насоса**: 10 секунд (в режиме AUTO)
- **Максимальная длительность насоса**: 60 секунд (предел безопасности, все режимы)
- **Выборка датчика**: 5 выборок с задержкой 10 мс каждая (~50 мс на чтение)
- **Скорость последовательного порта**: 115200 бод

## Переходы состояний

### Переходы режимов
- **AUTO → MANUAL**: Вызывается командой `PUMP_ON` или командой `MODE_MANUAL`
- **MANUAL → AUTO**: Вызывается командой `PUMP_OFF` или командой `MODE_AUTO`

### Переходы состояния насоса
- **OFF → ON**:
  - Режим AUTO: Влажность < `thresholdDry`
  - Режим MANUAL: Команда `PUMP_ON`
- **ON → OFF**:
  - Режим AUTO: Длительность >= `PUMP_DURATION_MS` ИЛИ влажность >= `thresholdWet`
  - Режим MANUAL: Команда `PUMP_OFF` ИЛИ длительность >= `PUMP_MAX_MS` (безопасность)

## Функции безопасности

1. **Максимальное время работы насоса**: Жесткий лимит 60 секунд предотвращает повреждение насоса
2. **Управление с гистерезисом**: Разные пороги предотвращают быстрое переключение насоса
3. **Сохранение в EEPROM**: Порог сохраняется после отключения питания
4. **Проверка ввода**: Команды проверяются перед выполнением
5. **Защита от переполнения буфера**: Буферы последовательного порта имеют ограничения размера

## Принципы проектирования

1. **Управление с гистерезисом**: Использует отдельные пороги для активации и деактивации
2. **Безопасность прежде всего**: Лимиты максимального времени работы предотвращают повреждение оборудования
3. **Двухрежимная работа**: Поддерживает как автоматическое, так и ручное управление
4. **Неблокирующий**: Операции последовательного порта и датчика не блокируют основной цикл
5. **Сохранение состояния**: Критические настройки сохраняются в EEPROM
6. **Модульная коммуникация**: Может работать с модулем ESP8266 или без него
